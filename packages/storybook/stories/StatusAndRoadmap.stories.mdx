## Note on Core vs. Commons vs. Labs

Please note that this Storybook contains references to components that exist in Core, Commons, and Labs. The primary motivation between separating these repos is to allow for rapid and atomic development of new components in labs without affecting the stability of components that are considered to be complete in Core and/or Commons.

The table below summarizes the differences between the two types of components side-by-side:

|                          | Core                                                                                                                                                                                                                                                                     | Commons                                                                                                                                                                                                                                                                     | Labs                                                                                                                                                                                                                                                                                                           |
| ------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Primary Purpose          | Core aims to assemble components that make up the "core" of a design system (e.g. buttons, modals, dropdowns, etc.)                                                                                                                                                      | Commons aims to create a secondary library that uses core components to address common use cases (e.g. info icon, accept/reject modal, etc.)                                                                                                                                | Labs aims to create a sandbox environment for new components currently being worked on, allowing individual components to be installed and upgraded separately in various Braze projects.                                                                                                                      |
| Safe for use in platform | âœ… Yes                                                                                                                                                                                                                                                                   | âœ… Yes                                                                                                                                                                                                                                                                      | âœ… Yes, breaking changes are always major version upgrades                                                                                                                                                                                                                                                     |
| Stability                | **Components in Core are considered stable** and will not have frequent breaking changes. We may add new features and bug fixes from time to time, however, most components here have been in use across the dashboard for a while, making them much more battle-tested. | **Components in Commons are considered stable** and will not have frequent breaking changes.
